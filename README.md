# Звіт до лабораторної роботи: Тестування програмного забезпечення

**Мета роботи:** Ознайомитися з методами автоматизованого тестування, використанням `assert` та аналізом покриття коду за допомогою Coverage.

---

## Виконання роботи

### 1. Перевірка assert та валідація
Було реалізовано власну перевірку введення даних з клавіатури за допомогою методу `input()` та інструкції `assert`. 
Також у класі `Figure` реалізовано валідацію аргументів: довжина має бути більшою за 0, а тип фігури — відповідати дозволеному списку.

### 2. Юніт-тести (Unittest)
У файлі `test.py` було створено тести для перевірки методів класу `Figure`.
* **Результат:** Тест `test_figure_length` провалився з помилкою `AssertionError: 3 != 'трикутник'`.
* **Причина:** У класі `Figure` властивість `get_figure_length` навмисно повертала `self.type` замість `self.length`.

### 3. Аналіз покриття (Coverage)
За допомогою бібліотеки `coverage` було згенеровано звіт про покриття коду:
* Команда `coverage run -m unittest test.py` зафіксувала виконання рядків коду.
* Команда `coverage html` створила візуальний звіт у форматі HTML.

## Висновки
* **Що зроблено:** Реалізовано механізми валідації в ООП та автоматизовані тести.
* **Результати:** Навмисна помилка була успішно виявлена тестами.
* **Складності:** Виникли труднощі з налаштуванням політики виконання скриптів у PowerShell та подвійним URL репозиторію, які було успішно вирішено.
